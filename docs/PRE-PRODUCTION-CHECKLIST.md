# 仙本那潜水 ERP — 投产前检查清单

投产前请按本清单逐项确认，并运行自动化冒烟测试。

---

## 一、环境与部署

- [ ] **MySQL**：已安装/已运行，数据库已创建，字符集 `utf8mb4`
- [ ] **后端 .env**：已配置 `DB_HOST`、`DB_USER`、`DB_PASSWORD`、`DB_NAME`、`JWT_SECRET`（生产环境务必修改默认密钥）
- [ ] **前端 API 地址**：生产环境 `VITE_API_BASE` 指向实际后端地址（如 `https://your-domain.com/api`）
- [ ] **低内存服务器前端部署**：若服务器内存仅 `2GB`，已在本地或 CI 完成前端构建，并确认服务器存在 `pure-admin-tpl/dist`
- [ ] **Nginx/反向代理**：若使用，已配置 `/api` 转发至后端、静态资源与 SPA 回退

---

## 二、数据与账号

- [ ] **初始化权限与用户**：已执行 `npm run init-permissions` 或等价初始化，确保存在管理员账号
- [ ] **默认账号**：确认登录账号（如 `admin` / `admin123`），投产后**立即修改密码**
- [ ] **基础数据**：如需预置房间、船只、员工等，已通过界面或 seed 脚本录入

---

## 三、自动化冒烟测试

**前提**：后端服务已启动（如 Docker 下 `backend` 容器运行在 3000 端口）。

```bash
# 在项目根目录
cd backend
npm run test:smoke
# 或指定后端地址（例如通过 Nginx 代理时）
node scripts/smoke-test.js https://your-domain.com/api
```

**脚本覆盖**：

| 模块     | 测试项 |
|----------|--------|
| 健康检查 | `GET /api/health` |
| 认证     | `POST /api/auth/login`、`GET /api/auth/me` |
| 公开报名 | `POST /api/enrollment/submit` |
| 学员     | 列表、创建、更新、（分配房间）、删除 |
| 房间     | 列表、创建、设置住客、删除 |
| 员工     | 列表、创建、删除 |
| 船只     | 列表、创建、删除 |
| 行程     | 创建（含多 DM/教练）、列表、更新、添加/移除参与者、删除 |
| 大屏     | `GET /api/display/rooms-status`、`dashboard-stats`、`trips-by-island` |

全部通过后再投产；若有失败，根据输出排查接口或数据后再测。

---

## 四、核心业务流程（建议人工走一遍）

1. **公开报名**  
   访问报名页 → 填写信息并提交 → 在「客人管理」中能看到新客人，且客人 ID 为短格式（如 G00001）。

2. **客人管理**  
   编辑客人（姓名、房间、学习内容、状态等）→ 保存成功；列表搜索、筛选、展开行查看行程记录正常。

3. **房间管理**  
   新建房间 → 分配住客 → 保存；列表与大屏房间状态一致（入住/空闲）。

4. **行程管理**  
   新建行程，选择船只、多选潜导与教练、选择客人 → 保存；列表与详情中多潜导/多教练显示正确；大屏展示今日/明日行程与房间状态。

5. **大屏/展示页**  
   在无登录或公开环境下打开展示页，房间状态、行程按岛屿/日期显示正确，刷新间隔符合预期（如 1 分钟）。

6. **登录与权限**  
   使用管理员账号登录 → 各菜单可访问；若有其他角色，验证其菜单与权限符合预期。

---

## 五、安全与性能建议

- [ ] 生产环境已更换默认 `JWT_SECRET` 与数据库密码
- [ ] 生产环境已关闭或限制不必要的调试日志
- [ ] 限流：后端已启用 `express-rate-limit`，可根据实际情况调整阈值
- [ ] HTTPS：若公网访问，已配置 SSL 证书
- [ ] 备份：已安排 MySQL 定期备份（建议每日）及备份恢复演练

---

## 六、上线后首次操作建议

1. 使用管理员账号登录后立即修改密码。
2. 再次执行一次冒烟测试（针对生产 API 地址），确认全部通过。
3. 用真实业务数据做 1～2 条完整流程（报名 → 入住 → 行程）验证。
4. 确认大屏在目标设备/浏览器上显示与刷新正常。
